üó∫Ô∏è K-Map Simplification

‚úÖ K-Map Rules
  • Group sizes must be powers of 2: 1, 2, 4, 8, 16

  • Don't care (X) can be 0 or 1 - use for larger groups

  • Uses combinational logic circuits
  • Very fast execution

  • Best for CISC processors

üöÄ Pipelining

‚úÖ 5-Stage MIPS Pipeline
  1. IF (Instruction Fetch): Fetch instruction from memory
  2. ID (Instruction Decode): Decode & read registers
  3. EX (Execute): ALU operation or address calc
  4. MEM (Memory Access): Load/Store data
  5. WB (Write Back): Write result to register
Pipeline Timing: Without pipeline: n instructions √ó k stages = n√ók cycles With pipeline: k + (n-1) cycles Speedup Formula: Speedup = (n √ó k) / (k + n - 1) For large n: Speedup ‚âà k (number of stages) Example: 100 instructions, 5 stages Without: 100 √ó 5 = 500 cycles With: 5 + 99 = 104 cycles Speedup = 500/104 ‚âà 4.8√ó

‚ö†Ô∏è Pipeline Hazards

üî¥ Data Hazard Types
  • RAW (Read After Write): True dependency - most common
  • WAR (Write After Read): Anti-dependency
  • WAW (Write After Write): Output dependency

Solution: Data forwarding (bypassing) - send result directly from EX stage to next instruction's EX stage without waiting for WB

üó∫Ô∏è Cache Mapping Techniques

1. Direct Mapping ‚ñº

Each memory block maps to exactly ONE cache line

Cache Line = Block Number mod (Number of Lines) Address breakdown: | Tag | Line | Offset | Example: 1024 blocks, 32 cache lines Block 65 ‚Üí Line 65 mod 32 = Line 1 Block 97 ‚Üí Line 97 mod 32 = Line 1 (conflict!)

‚úÖ Simple, fast lookup | ‚ùå High conflict misses

2. Fully Associative Mapping ‚ñº

Any block can go in ANY cache line

Address breakdown: | Tag | Offset | Must search ALL lines in parallel Uses Content Addressable Memory (CAM)

‚úÖ No conflict misses, best hit rate | ‚ùå Expensive, slow comparison

3. Set-Associative Mapping ‚ñº

Cache divided into sets, each set has N lines (N-way)

Set = Block Number mod (Number of Sets) Address breakdown: | Tag | Set | Offset | Example: 2-way set associative - Block can go in either of 2 lines in its set - 4-way and 8-way are common

‚úÖ Balanced: good hit rate with reasonable cost | Most common design

üìÑ Virtual Memory & Page Replacement

Virtual Memory Concepts
  • Page: Fixed-size block of virtual memory (4KB typical)
  • Frame: Physical memory block of same size
  • Page Table: Maps virtual pages to physical frames
  • Page Fault: Page not in RAM, must load from disk
  • TLB: Translation Lookaside Buffer - cache for page table entries

Page Replacement Algorithms

üì• FIFO

Replace oldest page first

  • Simple to implement
  • Suffers Belady's Anomaly
  • Not always efficient

üïê LRU

Replace least recently used

  • Better hit rate than FIFO
  • Needs tracking mechanism
  • Most practical choice

üîÆ Optimal (OPT)

Replace page not needed longest

  • Theoretical best
  • Impossible to implement
  • Used for comparison

Performance ranking (page faults): Optimal < LRU < FIFO
